Roadmap of the KAME Project (2002/4 - 2004/3)

$Id: roadmap-2002.html,v 1.5 2002/10/08 10:26:31 suz Exp $

Overview

The third phase of the KAME Project started in Apr 2002 and will end in Mar 2004. We divide the third phase into two Japanese years. The first Japanese year is for implementation as we have done in the first and second phase. The second Japanese year is used to feedback our results to each BSD project.
Apr 2002 - Mar 2003
Implement new features described below.
Apr 2003 - Mar 2004
Feedback our results to each BSD project.

IPv6

DNS server discovery
Finalize DNS server discovery ASAP.
DHCPv6
Implement DHCPv6 for DNS server and prefix delegation (PD) by IETF Yokohama.
ICMPv6 name lookup
Waiting for the final spec.
Source Specific Multicast (SSM)
Implement SSM by Sep 2002.
Multicast DNS
Too many spec changes, sigh. Hope that the spec will be finalized by Mar 2003.
Scoped address routing
Modify the current routing table to scope capable by Mar 2003.
X window system
Make X window system IPv6-friendly by Sep 2002 and contribute it to XF86 group.
VRRP
Implment VRRP by Mar 2003.

IPv6(low priority)

Router selection
If time allows, implement router selection by Mar 2003. This includes re-implementation of RA in application space.
ISATAP
If time allows, implement ISATAP by Mar 2003.

MobileIP6

MoblieIP6
Implement ID-17 by IETF Yokohama. ETSI in Sep 2002. Implement ID-1x by connectathon. Connectathon in Feb 2003.

IPsec

Revised IPsec architecture and security protocols
Follow them as soon as these specifications will be open.
IPsec policy
Document KAME's IPsec policy features and its API by Sep 2002. Design more general IPsec policy architecture with other projects and implement it by Mar 2003.
Key Management Protocol (KMP)
We are afraid tha spec of Key exchange will not be finalized by even Mar 2004. So, we will implement one simple key exchange protocol, JFK for example.
Minimum requirement spec of IPsec
Document minimum requirement spec of IPsec by Sep 2002.

Misc

The following items are included in the "snap" release but they are not supported by the KAME Project.
  • NAT/PT
  • SCTP
