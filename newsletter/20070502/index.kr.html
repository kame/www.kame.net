Type 0 Routing Header 이슈와 관련한 KAME Project의 활동

배경

CanSecWest2007에서 IPv6 Type 0 Routing Header가 공격수단으로 사용될 수 있다는 것을 지적한 'Fun with IPv6 Routing Headers'가 발제 되었다. 기술된 내용 중에는 IPv6 Type 0 Routing Header를 운용하는 어떤 두개의 IPv6노드(KAME/BSD기반 노드 포함, RFC2460 내용참조)를 공격하여 두 노드 사이의 Bandwidth를 차지하는 공격이 있다. 그러나 'attack'에 대한 프레젠테이션 내용은 몇 가지 기술적 결함이 있어 주된 위협은 여전히 존재하는듯 하다.

KAME Projcet 활동

KAME Project에서는 위와 같은 이슈를 해결하고 공격의 수단으로 사용되는 것을 방지하기 위하여 아래와 같은 활동을 수행했다.

  • Type 0 Routing header를 사용하지 못하게 KAME의 소스코드를 수정 ip6_rthdr0() 기능이 제거되어 Type 0 Routing Header는 unknwon Routing Header로 처리 net.inet6.ip6.hdrnestlimit 의 기본값은 15로 감소수정사항 확인
    (The proper diff can be found here.)
    cvsweb 수정사항: route6.c, in6_proto.c and ip6_input.c.
  • 다양한 BSD의 사용자와 KAME/BSD IPv6 Stack을 포함한 제품의 벤더들에게 이와 같은 문제 고지
  • IETF에서 이슈 해결을 위한 Spec 개선안 논의 (진행중)

권장 사항

이 문제와 관련하여 심각할 필요는 없다고 보고 있으나, 위와 같은 변경 사항을 적용하여 (하단에 있는 BSD의 보안관련 권고 내용 확인 요망) kernal을 reboot 할것을 권장한다. 특히 해당 IPv6 노드는 높은 bandwidth link에 잘 알려져 사용되고 있다.
