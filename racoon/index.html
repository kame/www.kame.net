What is Racoon ?

Racoon is the daemon which negotiates and configures a set of parameters of IPsec.
It is conformed with the following specifications at this moment:
  • RFC2407: The Internet IP Security Domain of Interpretation for ISAKMP.
  • RFC2408: Internet Security Association and Key Management Protocol (ISAKMP).
  • RFC2409: The Internet Key Exchange (IKE).
  • RFC2367: PF_KEY Key Management API, Version 2.
  • RFC2459: Internet X.509 Public Key Infrastructure Certificate and CRL Profile
  • RFC2535: Domain Name System Security Extensions
  • RFC2538: Storing Certificates in the Domain Name System (DNS)
  • RFC2104: HMAC: Keyed-Hashing for Message Authentication
  • draft-ietf-ipsec-ike-01.txt: The Internet Key Exchange (IKE)
  • draft-ietf-ipsec-ike-base-mode-02.txt: IKE Base Mode.
  • draft-ietf-ipsec-ike-modp-groups-04.txt: More MODP Diffie-Hellman groups for IKE.
  • draft-jenkins-ipsec-rekeying-06.txt: IPsec Re-keying Issues.
  • draft-ietf-ipsec-ike-lifetime-00.txt: Responder Lifetime Notify Message for IKE.
  • draft-ietf-ipsec-isakmp-gss-auth-06.txt: A GSS-API Authentication Method for IKE
  • draft-ietf-ipsec-pki-req-05.txt: A PKIX Profile for IKE

Where can I get ?

You can get the latest version from the CVS repository in the KAME stack which is a IPv6/IPsec stack.

How to install ?

You must confirm if OpenSSL 0.9.5aor later has been installed in the system because Racoon fully depends on it for all cryptographical processing. You can get OpenSSL from the www.openssl.org if you need it.

What is planning ?

ipsec WG is working out the new version of the key management protocol because IKE has security issues. There are some candidates as Son-of-IKE(SOI) for example, JFK(Just Fast Keying) or IKE v2. We are following them and we are going to implement one of them. some vendors have implemented XAUTH and ike-mode-cfg. we don't have any plan to implement both of them because they are not the internet standard.

Where can I ask ?

The Racoon users mailing list is a separate ML to discuss racoon-specific matters in general.

To subscribe to the list, send email with content subscribe to racoon-request@kame.net.
Past discussions can be found at http://www.kame.net/snap-users/mail-list.cgi or ftp://ftp.kame.net/pub/mail-list/racoon/.

Primary language for the mailing list is ENGLISH. If there's high demand we'll be able to maintain another list for Japanese language, so please do not hesitate to ask.
